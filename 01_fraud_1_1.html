Chapter 1 – Foundations: Python as an Analytical Instrument

1.1 Why We Are Learning Python This Way

You are not learning Python as an abstract programming language. You are learning Python as an analytical instrument — like a statistical calculator, but programmable, scalable, and capable of modeling adversarial systems.

Fraud analytics is fundamentally about:

  • Structured data
  • Probability and uncertainty
  • Classification decisions
  • Economic tradeoffs
  • Large-scale transaction streams

Python will become your modeling language for expressing these ideas.

In this first chapter, we focus on:

  • Basic syntax
  • Variables and data types
  • Arithmetic operations
  • Logical expressions
  • Control flow (if/else)

Every concept will be framed in a fraud context.

1.2 Variables – Representing Transaction Attributes

A variable stores information. In fraud analytics, variables represent measurable properties of a transaction. 

amount = 125.50
country = "US"
is_international = False

Data Types

  • int – integers (e.g., 100 transactions)
  • float – decimal numbers (e.g., $125.50)
  • str – text (e.g., country codes)
  • bool – True / False

1.3 Arithmetic – Measuring Risk Signals

Fraud models often transform raw variables. 

amount = 200
customer_income = 40000

ratio = amount / customer_income
print(ratio)

This ratio is more informative than the raw amount.

Basic Operators

  • + Addition
  • - Subtraction
  • * Multiplication
  • / Division
  • ** Exponentiation

1.4 Logical Expressions – Binary Decision Structure

Fraud detection systems ultimately produce binary decisions: approve or decline. 

amount = 1500

high_risk = amount > 1000
print(high_risk)

Comparison operators:

  • >
  • <
  • >=
  • <=
  • == (equal)
  • != (not equal)

Combining Conditions

amount = 1200
is_international = True

flag = amount > 1000 and is_international
print(flag)

Logical operators:

  • and
  • or
  • not
Conceptual Note: This is the primitive form of a rule-based fraud system. Before machine learning, fraud detection was entirely conditional logic. Understanding this structure is foundational.

1.5 Control Flow – Decision Logic

Now we formalize approve/decline decisions. 

amount = 1500
is_international = True

if amount > 1000 and is_international:
    decision = "DECLINE"
else:
    decision = "APPROVE"

print(decision)

Indentation matters in Python. It defines structure.

1.6 First Fraud Scoring Prototype

Let us build a primitive scoring function. 

score = 0

amount = 1200
is_international = True
customer_tenure = 1

if amount > 1000:
    score += 1

if is_international:
    score += 1

if customer_tenure < 2:
    score += 1

print("Risk Score:", score)

This is a linear additive risk model. It is conceptually similar to early credit bureau scorecards.

This scoring system:
  • Aggregates weak signals
  • Is transparent
  • Is interpretable
  • Is structurally linear
Later chapters will formalize this mathematically.

1.7 Mental Models to Internalize

  1. Every variable encodes information.
  2. Every rule expresses a hypothesis about fraud behavior.
  3. Risk scoring is structured aggregation.
  4. Binary decisions are thresholded scores.
